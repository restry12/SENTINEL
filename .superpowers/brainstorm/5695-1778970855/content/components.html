Estado Global — FireViewContext
// contexts/fire-view-context.tsx (evoluciona fire-selection-context)
interface FireViewState {
selectedFire: SelectedFireData // siempre hay uno seleccionado
expansionRange: '2h' | '6h' | '12h' // default '2h'
fires: SelectedFireData[] // todos los focos ordenados por FRP desc
viewMode: 'fire' | 'risk-zones' // preparado para futuro
}
interface FireViewActions {
selectFire: (id: string) => void
setExpansionRange: (range: '2h' | '6h' | '12h') => void
setViewMode: (mode: 'fire' | 'risk-zones') => void
}
REGLA CLAVE
Siempre hay un foco seleccionado. Al cargar, se auto-selecciona el de mayor FRP. No existe estado "sin selección". Esto elimina todos los null checks y estados vacíos.
Flujo de Datos
SentinelUpdate (socket)
→
FireViewContext
→
Componentes
FireViewContext consume useSentinel() internamente y expone los datos ya procesados:
  • Transforma fires[] → SelectedFireData[] con expansión calculada
  • Ordena por FRP desc
  • Auto-selecciona el top si no hay selección previa
  • Mantiene selección si el foco sigue en el nuevo update
